How to interpret the result

          
Use the bounded triage output to decide what happens next.


          

            allow
            
Low-risk event. Keep it logged for audit and later review, but no immediate escalation is needed.

          


          

            queue_for_review
            
Medium-risk event. A human reviewer should inspect the message and decide whether it is benign, suspicious, or needs escalation.

          


          

            escalate
            
High-risk event. Route it to the security lead or incident-response workflow for immediate human attention.

          


          

            Why the explanation matters
            
The explanation shows which suspicious features influenced the score. This makes the workflow easier to review, document, and improve over time.
